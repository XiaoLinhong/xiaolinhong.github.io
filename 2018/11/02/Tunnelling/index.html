网络隧道

本文字数： 547 | 阅读时长 ≈ 1 分钟

前言

从外部网络登录到局域网中的某台计算机(服务器)，常用的手段是用vpn，看起来不那么专业的手段有QQ远程，或者Teamview。
一种看起来更为高端的手段就是利用具有公网ip的服务器，建立一个网络隧道，或叫端口转化。
SSH提供远程登陆的一种方式，同时，所有传输的数据都是加密过的，SSH还提供了一种端口转化的功能。

端口转化： 可以将其他的协议（基于TCP）的传输数据，通过SSH转发，在转发的时候，还会提供额外的加密服务，这一过程也被成为隧道（Tunneling）。

作用

  • 加密其他协议的通信
  • 完成一些其他协议无法建立的TCP链接

使用

端口转发

ssh -L 8080:localhost:8888 203.23.41.5

将远程服务器（203.23.41.5）的8888端口服务，直接绑定在本地的8080端口上。

pqsql -P 8080 …直接在本地登录服务器上的数据库服务。

本地sshd会监听的数据库务端口8080

  1. 当数据库客服端去请求8080的服务时，本地的ssh服务会接到相关监听

  2. ssh客户程序会将该请求 转发给 远程服务器

  3. 远程服务器收到请求之后，会执行该请求

  4. 请求完成的数据，又返回去

默认的情况ssh绑定的是shell的服务（22端口，指向shell?）
