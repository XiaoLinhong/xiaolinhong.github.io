前言

Sphinx是一个以文本格式为基础的文档系统。 编写文档的时候，采用reST标记语言。可使用任何文本编辑器编写。编写完成后可转换为其他格式（包括编码格式 如PDF，和其他标记语言 如HTML），在使用相应的软件打开，阅读带有更好渲染的文档。

动因

  1. 可拜托Office
  2. 文本格式可以版本管理器有效监管
  3. Sphinx的文档系统可以转化为PDF格式和HTML格式，可直接托管至ReadTheDocs

安装

Sphinx
pip install sphinx

从代码注释生成API文档

2.6 基础配置

conf.py是一个python脚本，通过改变conf.py中的值，来控制Sphinx以怎样的方式处理文档。因为conf.py会被Sphinx执行，对于高级用户而言，可以用其来完成一些特殊任务，如扩展sys.path，添加python默认搜索路径，或导入库，用以识别文档的版本。

sphinx-quickstart产生的conf.py给出了大多数需要配置的参数，且用python语法添加了初始注释，用以说明相关参数的作用。 通常情况下只需要在给出的参数上进行修改即可，当然也可以通过添加额外的赋值语句来增加其他的配置参数。

需要注意的是，conf.py是一个python脚本，所有的改动需要遵循python语法，conf.py采用UTF-8编码，对于python2，在使用非ASCII时，需要使用Python Unicode字符，如value=”中文”。

2.7 自动归档

python代码中会自带较多文档（documentation strings），Sphinx支持通过autodoc扩展，从python的模块中导入docstrings。
要使用autodoc，需要在conf.py的 extensions列表（列表为python的内置数据类型）中添加’sphinx.ext.autodoc’。添加完成后，可以使用一些额外的指令，来完成自动归档。

段落是reST文档中最基本的语法块，段落通过一个或多个空行分割的一段文本内容。和Python一样, 对齐对于reST而言同样是一种有意义的操作，所以归属于同一段落的文本需要进行左对齐，并保持相同的缩进。
